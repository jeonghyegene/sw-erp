# 근무조 (WORK_SHIFT) 정책

> 파일럿 산출물 — STEP 6 정책 정의
> 적용 엔티티: [근무조 (WORK_SHIFT)](../entities/WORK_SHIFT_ENTITY.md) · 소유 화면: `ATT-WPL-001` 근무조 설정 탭
> 기준일: 2026-07-16 · 근거 소스: `assets/js/att-shift-data.js`, `assets/js/page-att-settings.js`, `assets/js/page-hr-employee.js`

이 문서는 근무조 마스터에 반복 적용되는 정책을 모은다. 특정 액션에서만 쓰는 검증은 [화면 액션 문서](../screens/ATT-WPL-001_ACTIONS.md)에, 상태 전이 조건은 [상태 문서](../states/WORK_SHIFT_STATE.md)에 둔다.

## POL-WS-01 근무조 코드 자동 채번

| 항목 | 내용 |
|---|---|
| 정책 목적 | 근무조 식별자를 일관 규칙으로 부여, 주간/야간 구분을 코드에 내재 |
| 적용 대상 | 신규 등록되는 모든 근무조 |
| 적용 조건 | 등록 시점 |
| 허용 처리 | `WT` + `D`(주간)/`N`(야간) + 일련번호 2자리 자동 부여. D/N은 심야(22~06) 겹침 여부로 자동 결정 |
| 금지 처리 | 사용자가 코드 직접 입력·수정 |
| 처리 결과 | 유일 코드 생성, 채번 후 불변 |
| 위반 시 표시 | (해당 없음 — UI에 코드 입력 필드 없음) |
| 백엔드 처리 | `추정` — 채번 유일성의 서버 보장은 확인 대상 |
| 예외 | 없음 |
| 확인 상태 | `확정` (`channelCode()` `:317`, `workTypeChar()` `:93`) |

## POL-WS-02 사용 중·사용 이력 근무조 변경 제한

| 항목 | 내용 |
|---|---|
| 정책 목적 | 근태 산정 기준의 무결성 보존 — 배치에 쓰인 근무조의 기준 시간이 바뀌면 과거 근태가 왜곡됨 |
| 적용 대상 | 사용 부서가 있거나(`inUse`) 과거 배치 이력이 있는(`everUsed`) 근무조 |
| 적용 조건 | 수정·미사용·삭제 시도 시 |
| 허용 처리 | 조회만. 변경하려면 부서·직원 배정을 먼저 해제 |
| 금지 처리 | 출·퇴근·휴게 등 산정 기준 수정, 미사용 처리, 삭제 |
| 처리 결과 | 액션 버튼 비활성 + 잠금 사유 안내 |
| 위반 시 표시 | 잠금 안내 한 줄("근무스케줄 배치에 사용된 이력이 있어 수정·삭제 불가" 등) + `사용 이력` 뱃지 |
| 백엔드 처리 | `확정`(프론트 가드) / 서버 재검증 `미결정` |
| 예외 | 색상 지정은 산정 기준이 아니므로 별도 확인 대상 |
| 확인 상태 | `확정` (`codeHasHistory` `:264`, `canEdit/canDelete` `:290~291`, 뱃지 `page-att-settings.js:283~286`) |

## POL-WS-03 전사 기본 근무조 유일성·보호

| 항목 | 내용 |
|---|---|
| 정책 목적 | 부서 신설 시 상속할 기준 근무조를 항상 1개 보장 |
| 적용 대상 | `isGlobalDefault=true` 근무조 |
| 적용 조건 | 상시 |
| 허용 처리 | 기본 근무조는 조회·기본 지정 변경 |
| 금지 처리 | 기본 근무조의 수정·미사용·삭제 |
| 처리 결과 | 전사 기본 근무조는 정확히 1개 유지 |
| 위반 시 표시 | `기본` 뱃지 + 액션 잠금 |
| 백엔드 처리 | `미결정` — 기본 근무조 교체 절차(현 기본 → 신규 기본) 정의 필요 |
| 예외 | 없음 |
| 확인 상태 | `확정`(유일성·잠금) / 교체 절차 `미결정` (`codeFlags` `:283`) |

## POL-WS-04 상태 변경 전자결재 대상 구분

| 항목 | 내용 |
|---|---|
| 정책 목적 | 근태 기준에 영향을 주는 변경만 승인 통제, 단순 추가는 신속 처리 |
| 적용 대상 | 근무조 상태 변경 액션 |
| 적용 조건 | 액션 유형별 |
| 허용 처리 | **추가 = 자유·결재 없음·이력 미기록**. **미사용·삭제 = 전자결재 상신 → 승인 후 반영**. **수정 = 결재 없음, 수정 이력만 기록**(부서 연결 근무조는 적용 시작일·사유 필수) |
| 금지 처리 | 미사용·삭제를 승인 없이 즉시 반영 |
| 처리 결과 | 미사용·삭제 상신 시 `승인대기` → 승인 시 반영, 변경 이력 기록(대상·사유·변경자·승인자) |
| 위반 시 표시 | 승인대기 뱃지, 중복 상신 차단 |
| 백엔드 처리 | `미결정` — 전자결재 연계·승인 트랜잭션 외부(현행 mock, 승인자 '김상무' 하드코딩) |
| 예외 | **재개(활성화)** — 주석은 결재 대상이나 코드는 즉시 반영 → `추정·구현 확인` |
| 확인 상태 | `확정`(추가/미사용/삭제 구분) / 재개 결재 여부·백엔드 `미결정` (`:213`, `submitCodeChange` `:977`, `approveCodeChange` `:987`, `activateCode` `:1003`) |

## POL-WS-05 근무조 선행 순서 (근무조 → 부서)

| 항목 | 내용 |
|---|---|
| 정책 목적 | 부서에 기본 근무조를 반드시 지정하도록 강제 |
| 적용 대상 | 부서 추가(`HR-EMP-001-M03`) |
| 적용 조건 | 부서 추가 시 |
| 허용 처리 | 근무조 마스터에 등록된 코드 중 선택 |
| 금지 처리 | 기본 근무조 미선택 상태의 부서 추가 |
| 처리 결과 | 근무조 코드가 선행 등록되어 있어야 부서 추가 가능 |
| 위반 시 표시 | 부서 추가 차단(인라인 검증) |
| 백엔드 처리 | `확정`(프론트 차단) |
| 예외 | 없음 |
| 확인 상태 | `확정` (`page-hr-employee.js:1360~1364`; 프로세스맵 §4.1·§5.1) |

> 이 정책은 부서 엔티티와 공유되는 관계 정책이다. 부서 관리 화면(`HR-EMP-001-M03`) 파일럿 시 부서 측에서도 참조한다.

## POL-WS-06 근태 산정 기준 계산 규칙

| 항목 | 내용 |
|---|---|
| 정책 목적 | 근무조 시간으로부터 근무·연장·심야·반차 구간을 일관 산출 |
| 적용 대상 | 모든 근무조 |
| 적용 조건 | 등록·수정 시 자동 재계산 |
| 허용 처리 | 총 근무 = 출~퇴 − 휴게. 연장 = 실근로 − 8시간(소정) 초과분. 심야 = 22:00~06:00 겹침. 야간조 = 심야 포함 시 자동. 반차 4h·반반차 2h 구간 산정 |
| 금지 처리 | 산정 규칙의 화면별 임의 변경 |
| 처리 결과 | `workHours`/`otMin`/`nightMin`/`isNight` 등 파생값 갱신 |
| 위반 시 표시 | 입력 시간 형식 오류는 인라인 검증 |
| 백엔드 처리 | `확정`(프론트 계산) / 서버 재계산 `미결정` |
| 예외 | 반차 구간에 휴게가 겹치면 근무 인정시간으로 치환(구간 미연장) — 정책 §4.10 |
| 확인 상태 | `확정` (`recompute()` `:80`, `nightMinutes()` `:56`, `halfDaySegments()` `:126`) |
