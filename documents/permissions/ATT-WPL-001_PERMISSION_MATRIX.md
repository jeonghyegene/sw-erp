## 2. 현행 권한으로 분리 어려운 액션 (권한 갭)

| 액션 | 갭 내용 | 제안 |
|---|---|---|
| WS-A05 미사용 / WS-A07 삭제 | 일반 정보 수정(Update)과 동일 권한으로 처리해도 되는지 불명확. 근태 기준에 영향을 주는 상태 변경이라 별도 통제가 자연스러움 | 미사용·삭제 **상신 권한**을 Update/Delete와 별도 플래그로 둘지 담당자 확정 |
| WS-A06 재개 | 결재 경유 여부가 미확정이라 권한 귀속도 미정 | 정책 POL-WS-04 확정 후 매핑 |
| WS-A08 결재 승인 | 현행 CRUD 축에 승인 권한이 없음 | 전자결재 승인 권한(역할 기반) 별도 정의 — 현행 권한 모델 밖 |

---

## 3. 데이터 범위·필드 제한 메모

- 근무조 마스터는 전사 단위 데이터로, 본인·부서 범위 구분이 없다(전사 전체 조회·관리). → 데이터 범위 세분화는 `미결정`.
- 필드 제한은 **사용 이력 있는 근무조의 산정 기준 필드 잠금**이 유일하게 확인됨(POL-WS-02). 마스킹 대상 민감 필드는 없음.

---

## 4. 확인 사항 종합

1. 근무조 설정 탭 접근 가능 **역할 범위**(인사/근태 담당 한정 여부) 확정.
2. 미사용·삭제 **상신 권한**을 일반 Update/Delete와 분리할지 (권한 갭 WS-A05·A07).
3. **전자결재 승인 권한**의 현행 권한 모델 편입 방식 (권한 갭 WS-A08).
4. 색상 지정(WS-A12)과 산정 기준 수정(WS-A04)의 권한 분리 여부.
5. 엑셀 다운로드 미제공이 의도인지(구현 갭) — 필요 시 Excel 권한 추가.
