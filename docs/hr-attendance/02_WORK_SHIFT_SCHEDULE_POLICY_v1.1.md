### 0.1 문서 목적

이 문서는 근무조 기준정보, 부서별 사용 가능 근무조, 부서별 기본 근무스케줄 편성, 직원별 일자 스케줄 및 사원유형별 근무조 변경 정책을 정의한다.

개발자는 본 문서의 조건, 처리 결과, 권한 및 우선순위를 화면·서버 검증·자동 처리·테스트의 공통 기준으로 사용한다.

### 0.2 적용 범위

- 근무조 기준정보 등록·수정·활성화·비활성화·삭제
- 부서와 사용 가능 근무조 연결·해제
- 부서별 1~5주차 기본 근무스케줄 편성
- 월별 근무스케줄 편성 및 직원별·일자별 변경
- 생산직 직접 변경과 사무직·연구직 전자결재 변경
- 입사·부서 이동·퇴사에 따른 스케줄 자동 처리
- 근무조 및 스케줄 변경 이력
- 예정 출근시간과 실제 출근시간의 불일치 표시

## 2. 조직 구조와 권한

### 2.1 조직 구조

- 조직 단위의 표준 명칭은 `부서`로 통일한다.
- 부서 유형은 본부, 팀, 파트이다.
- 허용 조직 구조는 다음과 같다.
  - 본부 > 팀 > 파트
  - 본부 > 파트

### 2.2 역할별 권한

| 역할 | 권한 |
|---|---|
| 인사담당자 | 근무조 등록·수정·활성화, 삭제·비활성화 신청, 부서-근무조 연결·해제, 부서별 스케줄 관리자 지정, 근무조 이력 및 전체 스케줄 변경 이력 조회 |
| 부서별 스케줄 관리자 | 담당 부서의 기본 근무스케줄 편성, 월별 근무스케줄 편성, 미래 스케줄 편집, 생산직 요청 변경, 관리 범위 내 스케줄 변경 이력 조회 |
| 직원 | 본인 스케줄 조회, 생산직 구두·메신저 변경 요청, 사무직·연구직 전자결재 변경 신청, 본인 출근시간 불일치 조회 |
| 대표이사 | 근무조 삭제·비활성화 이력 및 전체 스케줄 변경 이력 조회 |
| 시스템 | 상태 자동 산출, 월별 근무스케줄 편성 처리, 승인 결과 반영, 신청 만료, 임시 변경 종료 복귀, 입사·이동·퇴사 자동 처리, 출근시간 불일치 산출 |

### 2.3 부서별 스케줄 관리자 지정

| 규칙 ID | 정책 |
|---|---|
| ORG-001 | 한 부서에는 부서별 스케줄 관리자를 1명만 지정한다. |
| ORG-002 | 부서별 스케줄 관리자는 인사담당자만 지정하거나 변경할 수 있다. |
| ORG-003 | 관리자는 자신이 담당하는 부서의 직원만 직접 관리한다. |
| ORG-004 | 하위 부서에 별도 관리자가 없으면 상위 부서 관리자가 해당 하위 부서를 관리할 수 있다. |
| ORG-005 | 하위 부서에 관리자가 지정되면 해당 하위 부서는 그 관리자가 관리한다. |
| ORG-006 | 부서별 스케줄 관리자를 후임 없이 퇴사 처리할 수 없다. 후임 지정 전에는 퇴사 처리를 차단한다. |

## 4. 근무조 기준정보 정책

### 4.1 기본 입력 항목

| 항목 | 정책 |
|---|---|
| 근무조 코드 | 시스템 자동 생성 |
| 근무조명 | 필수, 중복 불가 |
| 출근시간 | 필수 |
| 퇴근시간 | 필수 |
| 휴게시간 | 0~2개 구간 |
| 운영상태 | 활성 또는 비활성 |

### 4.2 시간 및 휴게시간 검증

| 규칙 ID | 정책 |
|---|---|
| SFT-001 | 퇴근시간이 출근시간보다 이르면 다음 날 퇴근으로 판단한다. |
| SFT-002 | 하나의 근무조는 24시간을 초과할 수 없다. |
| SFT-003 | 휴게시간은 최대 2개 구간까지 등록할 수 있다. |
| SFT-004 | 각 휴게구간은 출근시간과 퇴근시간 사이에 있어야 한다. |
| SFT-005 | 두 휴게구간은 서로 겹칠 수 없다. |
| SFT-006 | 출근~퇴근 전체 시간이 4시간 미만이면 휴게시간을 등록하지 않는다. |
| SFT-007 | 출근~퇴근 전체 시간이 4시간 이상 8시간 미만이면 휴게시간 합계가 30분 이상이어야 한다. |
| SFT-008 | 출근~퇴근 전체 시간이 8시간 이상이면 휴게시간 합계가 1시간 이상이어야 한다. 1시간 미만만 차단하며, 1시간을 초과해도(예: 1시간 30분) 허용한다. |
| SFT-009 | 휴게시간을 2개로 나눈 경우 두 구간의 합산 시간으로 최소 휴게시간 충족 여부를 판정한다. |

### 4.3 중복 방지

| 규칙 ID | 정책 |
|---|---|
| SFT-010 | 근무조명은 활성·비활성 여부와 관계없이 중복 등록할 수 없다. |
| SFT-011 | 출근시간, 퇴근시간 및 등록된 모든 휴게구간이 동일한 근무조는 중복으로 판단한다. |
| SFT-012 | 비활성 근무조와 시간이 동일하더라도 신규 근무조를 생성할 수 없다. 필요한 경우 기존 근무조를 활성화한다. |

### 4.4 상태 정의

근무조의 운영상태와 사용상태를 하나의 상태로 합치지 않는다.

| 상태 구분 | 값 | 결정 방식 | 의미 |
|---|---|---|---|
| 운영상태 | 활성 | 인사담당자가 설정 | 부서 연결 후보로 사용할 수 있음 |
| 운영상태 | 비활성 | 승인 완료 시 시스템 처리 | 신규 부서 연결 및 스케줄 배정에 사용할 수 없음 |
| 사용상태 | 사용 중 | 시스템 자동 산출 | 현재 연결된 부서가 1개 이상 |
| 사용상태 | 미사용 | 시스템 자동 산출 | 현재 연결된 부서가 0개 |
| 승인 대기 여부 | 승인 대기 | 삭제·비활성화 결재 상신 시 | 관련 변경 및 신규 사용을 잠금 |

- `활성 + 미사용` 상태가 가능하다.
- 비활성 근무조는 활성화만으로 사용할 수 없다. 활성화 후 부서에 연결해야 한다.
- 사용상태는 현재 부서 연결 건수로 산출한다.

### 4.5 회사 공통 기본 근무조

|---|---|
| SFT-013 | 회사 공통 기본 근무조는 `09-18`로 관리한다. |

### 4.6 액션별 허용 조건

근무조 액션은 인사담당자만 수행한다. 아래 순서에 따라 허용 여부를 판정한다.

#### 4.6.1 공통 선행 판정

| 판정 항목 | 정책 |
|---|---|
| 부서-근무조 관계 | 하나의 근무조를 여러 부서에서 사용할 수 있고, 한 부서에도 여러 근무조를 연결할 수 있다. |
| 사용상태 산출 | 현재 근무조를 사용하는 부서 수를 기준으로 시스템이 자동 산출한다. |
| 회사 공통 기본 근무조 | `09-18` 근무조는 수정·삭제·비활성화할 수 없다. |
| 승인 대기 근무조 | 수정·활성화·삭제·비활성화, 부서 연결 및 미래 스케줄 신규 배정을 모두 차단한다. |

#### 4.6.2 액션 허용 판정표

4.6.1의 공통 제한을 먼저 확인한 후, 아래 `허용 조건`을 모두 충족해야 한다.

| 액션 | 허용 조건 — 모두 충족(AND) | 승인 | 처리 결과 |
|---|---|---|---|
| 등록 | 필수값·시간·휴게·중복 검증 통과 | 불필요 | 활성·미사용 근무조 생성 |
| 수정 | 미사용 · 근태 산정 이력 없음 · 오늘/미래 스케줄 배정 없음 | 불필요 | 근무조명·시간·휴게정보 변경 |
| 활성화 | 미사용 · 운영상태 비활성 | 불필요 | 운영상태를 활성으로 변경 |
| 비활성화 신청 | 미사용 · 오늘/미래 스케줄 배정 없음 | 필요 | 승인 대기로 전환하고 승인 완료 시 시스템이 자동 비활성화 |
| 삭제 신청 | 미사용 · 근태 산정 이력 없음 · 직원 스케줄 배정 이력 없음 | 필요 | 승인 대기로 전환하고 승인 완료 시 시스템이 데이터를 실제 삭제 |

- `사용 중`인 근무조는 수정·비활성화·삭제할 수 없다.
- 비활성화는 과거 근태 산정 이력과 과거 직원 스케줄 배정 이력이 있어도 가능하다.
- 삭제는 과거·오늘·미래 전체 기간에 직원 스케줄 배정 이력이 한 번도 없어야 한다.
- 활성화만으로는 부서에서 사용할 수 없다. 활성화 후 인사담당자가 부서에 연결해야 한다.
- 수정 조건의 `오늘/미래 스케줄 배정 없음`은 OI-004 결정 전까지 적용하는 차단 기준이다.

### 4.7 실제 삭제 정책

| 규칙 ID | 정책 |
|---|---|

## 6. 부서별 기본 근무스케줄 편성

### 6.1 편성 기준 구성

| TPL-005 | 부서별 기본 근무스케줄 편성 구조는 생산직·사무직·연구직에 동일하게 적용한다. |

### 6.2 월 주차 계산

- 한 주의 달력 기준은 월요일부터 일요일이다.
- 월별 근무스케줄 편성 대상은 월요일부터 금요일이다.
- 1주차는 해당 월의 첫 평일이 포함된 주이다.
- 월 시작 전 날짜와 다음 달 날짜는 포함하지 않는다.
- 휴일이 있어도 주차 번호를 다음 주로 밀지 않는다.
- 토요일·일요일 및 휴일은 월별 근무스케줄 편성 대상에서 제외한다.

예시: 월의 1일이 수요일인 경우

| 주차 | 월별 근무스케줄 편성 대상 |
|---|---|
| 1주차 | 1일 수요일~3일 금요일 |
| 2주차 | 6일 월요일~10일 금요일 |
| 3주차 | 13일 월요일~17일 금요일 |
| 4주차 | 20일 월요일~24일 금요일 |
| 5주차 | 27일 월요일~월말 |

월의 1일이 토요일 또는 일요일이면 다음 월요일부터 1주차를 시작한다.

### 6.3 편성 기준 변경 반영

## 9. 스케줄 처리 우선순위

동일 직원·일자에 여러 처리 원인이 존재할 때 다음 순서와 보호 규칙을 적용한다.

| 우선순위 | 처리 | 결과 |
|---:|---|---|
| 1 | 부서 이동 | 적용 중인 임시 변경을 종료하고 이동일부터 새 부서 기본 근무조를 적용 |
| 2 | 사무직·연구직 전자결재 승인 변경 | 일반 스케줄을 덮어쓰며 월별 근무스케줄 재편성으로부터 보호 |
| 3 | 월별 근무스케줄 편성 | 미래 일반 스케줄과 생산직 직접 변경 스케줄을 현재 부서의 기본 근무스케줄 편성 기준으로 다시 편성 |
| 4 | 생산직 요청 직접 변경 | 월별 근무스케줄 편성 이후 관리자가 선택한 미래 날짜에 직접 반영 |
| 5 | 부서별 기본 근무스케줄 편성 기준 | 별도 변경이 없는 직원·일자의 기본값 |
| 6 | 회사 공통 기본 근무조 `09-18` | 부서별 기본 근무스케줄 편성 정보가 없는 초기 상황의 최종 대체값 |

- 오늘과 과거 날짜에는 위 월별 재편성을 적용하지 않는다.
- 퇴사일 다음 날 이후에는 모든 미래 스케줄을 삭제한다.

| CAL-003 | 월별 근무스케줄 편성 대상 일자에는 부서별 기본 근무스케줄 편성 기준 또는 회사 공통 기본 근무조가 적용되므로 일자별 스케줄이 누락되지 않도록 한다. |

### 10.2 출근시간 불일치

예정 출근시간과 실제 출근시간의 차이는 절대값으로 계산한다.

```text
출근시간 차이(분) = |실제 출근시간 - 예정 출근시간|
```

| 규칙 ID | 정책 |
|---|---|
| CAL-004 | 출근시간 차이가 30분 이상이면 불일치로 표시한다. 정확히 30분인 경우도 포함한다. |
| CAL-005 | 조기 출근과 늦은 출근을 모두 같은 기준으로 판정한다. |
| CAL-006 | 30분 미만의 차이는 정상 범위로 처리한다. |
| CAL-007 | 불일치는 직원 본인, 해당 부서별 스케줄 관리자 및 인사담당자가 조회할 수 있다. |
| CAL-008 | 부서별 스케줄 관리자는 자신이 관리하는 부서와 별도 관리자가 없는 하위 부서의 불일치만 조회한다. |
| CAL-009 | 불일치 표시는 참고 정보이며 자동 급여 공제나 근태·급여 산정 차단에 사용하지 않는다. |
| CAL-010 | 화면에는 `30분 조기 출근`, `40분 늦은 출근`처럼 차이와 방향을 함께 표시한다. |

예시:

| 예정 출근 | 실제 출근 | 차이 | 판정 |
|---|---|---:|---|
| 10:00 | 09:45 | 15분 조기 | 정상 범위 |
| 10:00 | 09:31 | 29분 조기 | 정상 범위 |
| 10:00 | 09:30 | 30분 조기 | 불일치 |
| 10:00 | 09:00 | 60분 조기 | 불일치 |
| 10:00 | 10:30 | 30분 지연 | 불일치 |
