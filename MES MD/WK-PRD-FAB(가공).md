## 1) Summary

**One-liner**: 인쇄 후 제품 완성을 위한 후가공 공정을 기기 단위(단일형 또는 통합형)로 관리하며, 바코드 스캔 기반 순차 배정(큐)과 상태 추적을 수행한다.

### Goals

- 가공 기기별 작업 현황을 실시간으로 파악하고 순차 배정(큐)으로 작업 흐름을 관리한다
- 바코드 스캔 기반으로 작업자가 빠르게 작업을 시작·완료 처리할 수 있다
- 가공 유형별로 최적화된 화면 구조(단일 기기형/통합 다중 기기형)를 제공한다
- 상태 되돌리기(취소) 시 사유를 필수 선택하여 생산 이력을 추적한다

### Scope

**In Scope**
- **가공기기 목록 화면** (v1.3.0 신설): 가공 공정 내 모든 기기를 카드형으로 조회, 공정유형·위치·상태 필터, 담당 기기 우선 표시, 클릭 시 단일/통합 상세 이동 (DEC-25, DATASET 4)
- 가공 기기 단위 작업 현황 화면 (단일 기기형 / 통합 다중 기기형)
- 배정 작업 큐(대기) 관리 및 순차 작업 진행
- 바코드 스캔을 통한 작업 식별·시작·완료
- 작업 상태 관리: 가공대기 → 가공중 → 가공완료 / 상태 되돌리기(취소) / 작업중지 (POL-PRD-120) ※ 보류 폐기(v1.10.0) — 일시 유예는 작업중지로 대체
- **가공 유형별 독립 상태 관리**: 품목 1건에 대해 코팅/박/제본/오시/접지/타공/형압 등 각 가공유형이 독립된 가공대기/가공중/가공완료 상태를 가짐 (POL-PRD-121)
- **가공 유형 간 순서 의존성 적용**: 코팅→박→재단 등 선행 가공 완료 조건 검증 (POL-PRD-122)
- **불량 표기 및 재작업 복귀**: 가공 중 불량 발견 시 해당 가공유형만 불량 전환, 재작업 확정 시 해당 가공유형만 가공대기로 복귀 (POL-PRD-123, POL-PRD-124)
- 기기 상태 변경 (가동중/일시정지/오프라인·점검중)
- 상태 되돌리기 사유 기록 (재작업/기기오류/바코드오류/잘못누름 4종)
- 작업이력 조회
- 완료취소 권한 차등 (담당자=당일 건, 관리자=제한 없음)
- **가공 유형별 독립 작업중지**: 특정 가공유형 단위 작업중지/해제, 승인 불필요·사유 필수, 해제 시 직전 상태 복귀 (POL-PRD-086, POL-PRD-082~085)
- **감리 대상 가공유형 트리거**: 형압 등 감리가 요구되는 가공유형 완료 시 WK-PRD-AUD로 감리접수 TRIGGER, 완료취소 시 감리롤백 TRIGGER (POL-MES-010)
- 감사 로그 기록

**Out of Scope**
- 작업 삭제 (POL-MES-005를 따른다)
- 가공 배정 기준 설정 (접수시스템 또는 그룹웨어 담당)
- 가공 마스터 코드 관리 (시스템-코드관리 담당)
- 포장 공정 처리 (WK-PRD-PKG에서 담당)
