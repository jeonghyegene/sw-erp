## 1) Summary

**One-liner**: 각 공정(인쇄/출력/가공/포장)의 작업 데이터를 **조회기간 × 일자(行)** 축으로 **공정별 매수·작업건**을 집계하여, 작업자·관리자가 현 작업과 무관하게 독립 윈도우 팝업으로 조회하는 **조회 전용** 화면이다.

### Goals

- 공정별 일자 단위 작업 매수(수량)·작업건을 한 화면에서 파악한다
- 조회기간(기본 최근 7일) 동안 해당 공정 기기 전체의 작업 실적을 집계한다
- 기준(인쇄/출력/가공/포장)에 따라 컬럼·KPI 라벨이 자동 전환되는 단일 공통 포맷을 제공한다
- 현재 작업과 분리된 독립 팝업으로 병행 조회를 지원한다

### Scope

**In Scope**
- WST-01 작업통계 그리드 (일자별 행 + sticky 합계행, 공정별 컬럼 상이)
- 상단 KPI 요약 (총 매수 · 총 작업건)
- 필터 (조회기간 / 기기(호기) / **작업자(LR-016)** / 기준(공정) 토글 / 초기화 / 조회 / 엑셀출력)
- WST-02 **일자 드릴다운 모달** (일자 행 클릭 → 해당 일자 작업번호 목록, LR-017)
- 조회기간 기본값(최근 7일)·상한(6개월) 검증
- 각 공정 작업건 리스트(기기 상세) 화면의 '작업통계' 진입 버튼 + 상세 진입 시 해당 기기 프리필터
- 엑셀 다운로드 + 감사 로그 (POL-ADM-031 / POL-DATA-004)
- 독립 윈도우 팝업 호출 (window.open, GNB·LNB 비노출)

**Out of Scope**
- 작업 상태 변경·수량 입력·완료 처리 → 각 공정 작업 화면 [교차참조: MES / WK-PRD-PRC · WK-PRD-FAB · WK-PRD-PKG]
- 작업 수량 산정 기준·완료 판정·매수(정매/여분/추가) 정의 → 각 공정 Work MD
- 기기 가동률·속도·표준시간 등 기기 KPI 집계 → [교차참조: MES / WK-PRD-MON]
- 재작업 건수·재작업액 집계 → [교차참조: MES / WK-PRD-RWK]
- 월 6개월 초과·연 단위 장기 추이·라인 차트 → 본 MD 범위 외(필요 시 MON 가동현황)

## 2) Decisions

| ID | 결정 사항 | 근거 |
|----|-----------|------|
| DEC-001 | 작업통계는 **조회 전용(READ)** 이며 작업 데이터 수정·수량 입력·완료 처리 기능을 부여하지 않는다 | POL-AUTH-001 기본 차단, SoT 단일성 (작업 입력은 각 공정 소관) |
| DEC-002 | 인쇄/출력/가공/포장 4공정을 **단일 공통 화면(셸) + 공정 기준 파라미터(proc)** 로 처리한다. 기준에 따라 컬럼·KPI 라벨·집계식만 전환된다 | 사용자 확정 ("공통 포맷, 기준에 따라 달라"). MON 타입별 컬럼 준용 |

| LR-009 | 상단 KPI는 인쇄/출력/가공 = 총 매수(총 인쇄매수/총 출력판수/**가공=총 인쇄매수**) + 총 작업건 2종, **포장 = 총 포장건수 1종**을 표시한다. DEC-009 |
| LR-010 | 집계 대상 데이터(작업·수량·완료 판정)는 각 공정 작업 엔티티를 READ 한 결과이며, 본 MD는 작업 수량 산정·완료 판정 산식을 재정의하지 않는다. [교차참조: MES / WK-PRD-PRC · WK-PRD-FAB · WK-PRD-PKG] |
| LR-011 | **인쇄 매수 정의** — 일자별 총합 = Σ(정매 + 여분 + 추가매수), KPI "총 인쇄매수" = 기간 Σ총합. 컬럼은 정매·여분·추가매수·총합으로 분해 표시한다. [교차참조: MES / WK-PRD-PRC §6 PrintJob(정매/여분/추가매수)] |
| LR-012 | **출력(CTP) 매수 정의** — 일자별 총합 = Σ조정수량(정매수량 + 추가사용 반영 후 확정값), KPI "총 출력판수" = 기간 Σ조정수량. 컬럼은 정매수량·추가사용·조정수량으로 표시한다. [교차참조: MES / WK-PRD-PRC §6 CtpJob(정매수량/추가사용수량/조정수량)] |
| LR-013 | **가공 매수 정의** (v1.1.0 사용자 확정 2026-06-05) — 컬럼은 **인쇄매수·불량매수**로 표시한다(완료수량→인쇄매수·불량수량→불량매수 라벨 변경, **양품수량 컬럼 삭제**). 일자별 인쇄매수 = Σ인쇄매수, KPI "총 인쇄매수" = 기간 Σ인쇄매수. 데이터 원본은 FabJob 완료수량/불량수량이며 표시 라벨만 변경한다(SoT는 FAB). [교차참조: MES / WK-PRD-FAB §6 FabJob(완료수량/불량수량)] |
| LR-014 | **포장 건수 정의** (매수 아님 — 사용자 확정 2026-06-04) — 컬럼은 **포장완료(건)·취소(건)·포장건수**로 표시한다. 포장완료(건)=일자별 완료 처리된 포장 작업 수(상태 포장완료/출고대기 등 완료), 취소(건)=일자별 취소된 포장 작업 수(상태 취소), 포장건수=포장완료+취소(일자 처리 총건수). KPI "총 포장건수"=기간 Σ포장건수. 포장은 매수(정매/여분) 단위가 아닌 **건수** 단위로 집계하며 별도 '작업건' 컬럼을 두지 않는다. (통·출고건 미사용 — OI-002 철회). [교차참조: MES / WK-PRD-PKG §6 PackageJob(상태: 포장완료/취소)] |
| LR-015 | 엑셀출력 컬럼 구조는 화면 그리드와 동일하며, 권한은 POL-ADM-031, 다운로드 감사 로그는 POL-DATA-004를 따른다. DEC-011 |
| LR-016 | **작업자 필터** (DEC-012) — 상단 '작업자' 필터로 특정 작업자의 작업만 한정 집계한다. 기본값 '전체'. 작업자 옵션은 **해당 공정 기기의 담당자목록**(기기↔작업자 배정)에서 distinct 산출한다. 작업자 정보가 없는 공정(예: 출력 — 담당자목록 미지정 기기)은 필터를 비활성화한다. 작업자별 집계의 원천은 각 공정 작업 엔티티의 '작업자' 필드다. [교차참조: MES / WK-PRD-PRC·FAB·PKG 작업자 필드] |
| LR-017 | **일자 드릴다운 모달(WST-02)** (DEC-013) — 그리드 일자 행 클릭 시 해당 일자의 **작업번호 단위 목록 모달**을 표시한다. 컬럼: 작업번호·기기·작업자·품목명·고객명·상태 + 공정별 측정값(포장은 측정 컬럼 대신 상태로 포장완료/취소 구분). 적용된 기기·작업자 필터를 반영하며, 모달 합계는 해당 일자 작업건·총 매수(포장=총 포장건수)와 그리드 행이 정합해야 한다. RWK 상세 모달 패턴 재사용. [교차참조: MES / WK-PRD-RWK LR-013] |
