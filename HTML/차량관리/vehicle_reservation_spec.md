# 사내 차량 예약 관리 페이지 구축 시나리오 및 화면 정의서

## 1. 문서 개요

본 문서는 **스케줄러 기반 사내 차량 예약 및 반납 관리 페이지** 구축을 위한 시나리오와 화면 정의를 정리한 문서이다.  
목표는 차량 예약부터 운행 전 점검, 운행 후 반납, 주행거리 보정, 관리자 알림까지의 흐름을 하나의 업무 프로세스로 통합하는 것이다.

---

## 2. 구축 목표

사내 공용 차량 운영 시 아래 문제를 해결한다.

- 차량 예약과 실제 운행 정보가 분리되어 관리되는 문제
- 운행 전/후 계기판 사진 누락으로 인한 데이터 신뢰도 저하
- 이전 운행 종료거리와 다음 운행 시작거리 불일치 시 추적 어려움
- 반납 대상 차량이 있음에도 후속 처리가 누락되는 문제
- 관리자 확인이 필요한 예외 상황(거리 갭 발생)에 대한 즉시 인지 부족

이를 위해 예약 화면, 운행 시작/종료 처리 화면, 차량 반납 화면, 관리자 모니터링 화면을 구성한다.

---

## 3. 업무 범위

### 3.1 사용자 기능
- 사용일자, 목적지, 차량 선택 후 차량 예약
- 운행 시작 전 출발 거리 확인 및 사진 등록
- 이전 사용자의 도착 거리를 자동으로 출발 거리로 세팅
- 실제 계기판 값과 자동 세팅 값이 다를 경우 수정 가능
- 운행 종료 후 최종 도착 거리 입력 및 사진 등록
- 반납 처리할 차량이 있는 경우 반납 화면 진입

### 3.2 관리자 기능
- 차량별 예약 현황 조회
- 거리 갭 발생 건 알림 수신
- 거리 갭 내역 검토 및 승인/확인
- 차량별 운행 이력 및 사진 증빙 조회
- 반납 누락/지연 상태 확인

---

## 4. 핵심 요구사항 정리

### 4.1 예약 정보
사용자는 아래 항목을 선택하여 예약한다.

- 사용일자
- 목적지
- 차량

### 4.2 운행 전 처리
- 선택한 차량의 **직전 운행 기록의 도착 거리**를 다음 운행의 **출발 거리**로 자동 세팅한다.
- 운행 전 **계기판 사진 등록은 필수**이다.
- 사용자는 실제 계기판 값을 확인 후 출발 거리를 수정할 수 있다.

### 4.3 운행 후 처리
- 운행 종료 시 **최종 도착 거리 입력은 필수**이다.
- 운행 후 **계기판 사진 등록은 필수**이다.

### 4.4 거리 갭 처리
예시:
- 실제 계기판: 105km
- ERP상 직전 종료거리: 100km

처리 방식:
1. 시스템은 직전 종료거리 100km를 출발 거리로 자동 세팅한다.
2. 사용자가 실제 계기판 기준으로 출발 거리 105km로 수정한다.
3. 시스템은 **거리 갭 5km 발생**으로 판단한다.
4. 설정된 허용 범위를 초과하거나 예외 발생 조건에 해당하면 **관리자 알림을 발송**한다.
5. 해당 건은 차량 관리 화면과 관리자 모니터링 화면에서 추적 가능해야 한다.

### 4.5 차량 반납 처리
- 차량 예약 후, 사용자가 이전에 사용한 차량 중 **반납 미처리 차량**이 있으면 예약 완료 후 **"차량 반납" 버튼을 노출**한다.
- 버튼 클릭 시 **이전 예약 차량에 대한 반납 처리 화면**으로 이동한다.
- 반납일은 자동 세팅되며 사용자가 수정 가능하다.
- 반납 처리 시 **도착 거리 입력** 및 **사진 등록은 필수**이다.

## 5. 구축 시나리오

## 5.1 전체 프로세스 시나리오

```text
차량 예약
  → 예약 완료
    → 반납 대상 차량 존재 여부 확인
      → 있으면 "차량 반납" 버튼 노출
      → 없으면 예약 상세 화면 이동
  → 예약일 도래
    → 운행 시작 처리
      → 이전 도착 거리 자동 세팅
      → 출발 거리 확인/수정
      → 운행 전 사진 등록
      → 거리 갭 체크
        → 허용 범위 초과 시 관리자 알림
  → 운행 종료 처리
    → 최종 도착 거리 입력
    → 운행 후 사진 등록
    → 기록 저장
  → 필요 시 차량 반납 처리
    → 반납일 자동 세팅
    → 도착 거리 입력
    → 사진 등록
    → 반납 완료
```

## 5.2 상세 시나리오

### 시나리오 A. 차량 예약
1. 사용자가 차량 예약 페이지에 접속한다.
2. 사용일자, 목적지, 차량을 선택한다.
3. 시스템은 해당 차량의 예약 가능 여부를 확인한다.
4. 예약 가능 시 예약을 등록한다.
5. 예약 완료 후 사용자의 미반납 차량 여부를 조회한다.
6. 미반납 차량이 있으면 예약 완료 팝업 또는 상세 화면에 **"차량 반납" 버튼**을 표시한다.

### 시나리오 B. 운행 시작 처리
1. 사용자가 예약 건 상세에서 운행 시작 버튼을 누른다.
2. 시스템은 해당 차량의 최근 종료 운행 기록을 조회한다.
3. 최근 종료 기록의 도착 거리를 현재 예약의 출발 거리 기본값으로 세팅한다.
4. 사용자는 계기판 실물 확인 후 출발 거리를 검토한다.
5. 사용자는 운행 전 계기판 사진을 등록한다.
6. 자동 세팅 거리와 실제 입력 거리 차이를 계산한다.
7. 차이가 허용 범위를 초과하거나 관리 대상 조건이면 관리자에게 알림을 보낸다.
8. 운행 시작 정보를 저장한다.

### 시나리오 C. 운행 종료 처리
1. 사용자가 운행 종료 버튼을 누른다.
2. 최종 도착 거리를 입력한다.
3. 운행 후 계기판 사진을 등록한다.
4. 시스템은 입력값 유효성을 검증한다.
5. 운행 종료 정보를 저장한다.
6. 저장된 도착 거리는 이후 다음 사용자 출발 거리 기본값으로 사용된다.

### 시나리오 D. 거리 갭 발생 처리
1. 이전 종료거리가 100km로 저장되어 있다.
2. 다음 사용자가 운행 시작 화면 진입 시 출발 거리 기본값은 100km로 표시된다.
3. 사용자가 실제 계기판 확인 후 105km로 수정 입력한다.
4. 시스템은 `실입력 출발 거리 - 직전 종료 거리 = 5km`로 계산한다.
5. 5km가 허용 범위 초과 시 관리자에게 다음 정보를 포함한 알림을 보낸다.
   - 차량명
   - 이전 예약자
   - 이전 종료거리
   - 현재 예약자
   - 실제 입력 출발거리
   - 갭 거리
   - 등록 시각
6. 관리자 화면에는 예외 상태로 표시한다.

### 시나리오 E. 차량 반납 처리
1. 사용자가 신규 예약 완료 후 미반납 차량이 있음을 확인한다.
2. "차량 반납" 버튼을 클릭한다.
3. 시스템은 미반납 차량 정보를 반납 처리 화면에 표시한다.
4. 반납일은 오늘 날짜로 자동 세팅한다.
5. 사용자는 필요 시 반납일을 수정한다.
6. 최종 도착 거리 입력 및 계기판 사진을 등록한다.
7. 반납 완료 처리 후 상태를 "반납 완료"로 변경한다.

## 6. 화면 구성도

1. 차량 예약 화면
2. 예약 상세/운행 처리 화면
3. 운행 시작 처리 팝업 또는 화면
4. 운행 종료 처리 팝업 또는 화면
5. 차량 반납 처리 화면
6. 차량 관리 화면
7. 관리자 알림/예외 처리 화면
8. 차량 운행 이력 조회 화면

## 7. 화면 정의서

## 7.1 차량 예약 화면

### 화면 목적
사용자가 차량 사용 일정을 예약하는 화면

### 주요 항목
| 항목 | 타입 | 필수 | 설명 |
|---|---|---:|---|
| 사용일자 | Date | Y | 차량 사용 예정 일자 |
| 목적지 | Text / Select | Y | 방문 목적지 또는 이동 목적 |
| 차량 | Select | Y | 예약 가능한 차량 목록 |
| 사용 목적 | TextArea | N | 출장/외근/납품 등 상세 내용 |
| 예약 버튼 | Button | - | 예약 등록 |

### 기능 정의
- 선택한 날짜 기준 차량 예약 가능 여부 조회
- 동일 시간대 중복 예약 방지
- 예약 완료 후 미반납 차량 존재 시 "차량 반납" 버튼 노출

### 예외 처리
- 예약 가능 차량이 없으면 안내 메시지 노출
- 필수 항목 미입력 시 저장 불가

---

## 7.2 예약 상세 / 운행 처리 화면

### 화면 목적
예약 건의 상세 정보 조회 및 운행 시작/종료 처리 수행

### 주요 항목
| 항목 | 타입 | 설명 |
|---|---|---|
| 예약번호 | Text | 예약 식별값 |
| 사용자명 | Text | 예약자 |
| 사용일자 | Text | 예약 일자 |
| 목적지 | Text | 예약 목적지 |
| 차량명 | Text | 예약 차량 |
| 예약 상태 | Badge | 예약완료 / 운행중 / 종료 / 반납대기 / 반납완료 |
| 운행 시작 버튼 | Button | 운행 시작 화면 진입 |
| 운행 종료 버튼 | Button | 운행 종료 화면 진입 |
| 차량 반납 버튼 | Button | 미반납 차량 존재 시 노출 |

### 기능 정의
- 상태값에 따라 버튼 노출 제어
- 운행 시작 전에는 종료 버튼 비활성화
- 미반납 차량 존재 여부에 따라 차량 반납 버튼 표시

---

## 7.3 운행 시작 처리 화면

### 화면 목적
운행 시작 시 출발 거리 및 증빙 사진을 등록하는 화면

### 주요 항목
| 항목 | 타입 | 필수 | 설명 |
|---|---|---:|---|
| 차량명 | Text | - | 대상 차량 |
| 직전 종료 거리 | Number | - | 이전 운행자의 최종 도착 거리 |
| 자동 세팅 출발 거리 | Number | - | 직전 종료 거리 기준 기본값 |
| 실제 출발 거리 | Number | Y | 현재 계기판 기준 출발 거리 |
| 운행 전 계기판 사진 | File/Image | Y | 사진 업로드 필수 |
| 거리 갭 | Calculated | - | 실제 출발 거리 - 직전 종료 거리 |
| 갭 사유 | TextArea | N | 필요 시 사용자 메모 |
| 저장/운행 시작 버튼 | Button | - | 시작 처리 저장 |

### 기능 정의
- 직전 종료 거리를 기본값으로 자동 세팅
- 실제 출발 거리 수정 가능
- 저장 시 거리 갭 자동 계산
- 갭 허용 범위 초과 시 관리자 알림 발송

### 검증 규칙
- 실제 출발 거리 미입력 시 저장 불가
- 운행 전 계기판 사진 미등록 시 저장 불가
- 실제 출발 거리는 0 이상 숫자만 허용
- 직전 종료 거리보다 작은 값 입력 시 경고 또는 저장 제한 정책 적용

---

## 7.4 운행 종료 처리 화면

### 화면 목적
운행 종료 시 최종 도착 거리 및 증빙 사진 등록

### 주요 항목
| 항목 | 타입 | 필수 | 설명 |
|---|---|---:|---|
| 차량명 | Text | - | 대상 차량 |
| 출발 거리 | Number | - | 시작 처리 시 저장된 값 |
| 최종 도착 거리 | Number | Y | 운행 종료 시 계기판 값 |
| 운행 후 계기판 사진 | File/Image | Y | 사진 업로드 필수 |
| 운행 메모 | TextArea | N | 특이사항 기록 |
| 저장/운행 종료 버튼 | Button | - | 종료 처리 저장 |

### 기능 정의
- 운행 종료 시 최종 도착 거리 저장
- 종료 처리 완료 후 예약 상태를 종료로 변경
- 저장된 최종 도착 거리를 다음 예약 시작 기본값 후보로 사용

### 검증 규칙
- 최종 도착 거리 미입력 시 저장 불가
- 운행 후 사진 미등록 시 저장 불가
- 최종 도착 거리가 출발 거리보다 작을 경우 저장 제한 또는 관리자 검토 상태 전환

---

## 7.5 차량 반납 처리 화면

### 화면 목적
이전 예약 차량의 반납 상태를 마감 처리하는 화면

### 주요 항목
| 항목 | 타입 | 필수 | 설명 |
|---|---|---:|---|
| 대상 차량 | Text | - | 반납 처리 차량 |
| 반납일 | Date | Y | 기본값 오늘 날짜, 수정 가능 |
| 최종 도착 거리 | Number | Y | 반납 시 계기판 값 |
| 계기판 사진 | File/Image | Y | 반납 증빙 사진 |
| 반납 메모 | TextArea | N | 상태/이상 여부 기록 |
| 반납 완료 버튼 | Button | - | 반납 처리 저장 |

### 기능 정의
- 신규 예약 직후 또는 마이페이지/상세화면에서 진입 가능
- 반납일 자동 세팅
- 반납 후 상태를 반납 완료로 변경

### 검증 규칙
- 반납일, 도착 거리, 사진 누락 시 저장 불가

---

## 7.6 차량 관리 화면

### 화면 목적
차량별 현재 상태, 최근 운행 거리, 예외 상황을 관리자가 확인하는 화면

### 주요 항목
| 항목 | 설명 |
|---|---|
| 차량 목록 | 차량별 상태 조회 |
| 현재 상태 | 예약가능 / 예약중 / 운행중 / 반납대기 |
| 최근 종료 거리 | 마지막 운행 종료 거리 |
| 최근 종료 사진 | 마지막 계기판 사진 썸네일 |
| 갭 발생 여부 | 거리 불일치 여부 |
| 미반납 여부 | 반납 지연 상태 |
| 상세보기 | 운행 이력 및 예외 이력 진입 |

### 기능 정의
- 차량 상태 실시간 조회
- 갭 발생 차량 강조 표시
- 관리자 조치 필요 건 필터링

| 차량명 | 대상 차량 |
| 이전 종료 거리 | 직전 기록 |

### 기능 정의
- 거리 갭 발생 즉시 관리자 알림 발송
- 예외 건 상태 변경 및 의견 기록
- 차량별/사용자별 반복 예외 통계 조회 가능

### 알림 채널 예시
- 웹 알림
- 이메일
- 메신저(사내 협업툴)

## 9. 주요 비즈니스 룰

### 9.1 거리 자동 세팅 룰
- 동일 차량의 가장 최근 정상 종료 기록의 도착 거리를 다음 출발 거리 기본값으로 사용한다.
- 정상 종료 기록이 없으면 수동 입력하도록 한다.

### 9.2 거리 갭 계산 룰
- `거리 갭 = 실제 출발 거리 - 직전 종료 거리`
- 거리 갭이 0이면 정상
- 거리 갭이 허용 범위 이내이면 저장 허용 + 로그 기록
- 거리 갭이 허용 범위 초과이면 저장 허용 여부는 정책에 따라 결정하되 관리자 알림 필수

### 9.3 갭 허용 범위 정책 예시
- 기본 허용 범위: 0~3km
- 4km 이상: 관리자 알림
- 음수 입력: 비정상 데이터로 간주하여 저장 제한 또는 검토 상태 처리

> 허용 범위는 운영 정책에 따라 관리자 설정값으로 분리하는 것을 권장한다.

### 9.4 사진 증빙 룰
- 운행 시작, 운행 종료, 차량 반납 처리 단계에서 사진 첨부는 필수
- 사진 없는 저장은 불가
- 이미지 파일 형식 및 최대 용량 제한 필요

### 9.5 반납 버튼 노출 룰
- 로그인 사용자의 미반납 차량 건이 1건 이상 존재하면 노출
- 신규 예약 완료 후 우선 노출
- 마이페이지 및 예약 상세에서도 동일하게 접근 가능하도록 구성 권장

## 11. 권장 UI/UX 포인트

- 예약 화면에서는 차량 가용 상태를 달력/스케줄 형태로 직관적으로 제공
- 운행 시작/종료 화면에서는 숫자 입력보다 **이전 값 대비 차이**를 시각적으로 강조
- 거리 갭 발생 시 저장 전 경고 메시지와 함께 관리자 알림 예정 문구 노출
- 사진 업로드 후 즉시 썸네일 미리보기 제공
- 차량 반납 버튼은 상단 고정 액션 버튼으로 노출하여 누락 방지
- 관리자 화면은 예외 건 우선 정렬 및 색상 강조 적용

## 14. 결론

본 시스템은 단순 차량 예약 기능이 아니라, **예약 → 운행 시작 → 운행 종료 → 반납 → 예외 알림**까지 연결되는 차량 운영 관리 체계로 설계되어야 한다.  
특히 **직전 종료 거리의 자동 승계**, **사진 기반 증빙 의무화**, **거리 갭 발생 시 관리자 알림**, **미반납 차량의 후속 처리 유도**가 핵심 설계 포인트이다.

이 기준으로 화면과 데이터 구조를 설계하면 실제 운영 중 발생하는 누락, 오입력, 책임 추적 문제를 효과적으로 줄일 수 있다.
